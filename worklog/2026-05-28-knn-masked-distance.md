\# 2026-05-28 — KNN 측위 개선: 마스킹 거리 (dropout 환경 강건)



\## 한 일

\- `core/knn.py` 거리 계산을 \*\*마스킹 거리(measured-only distance)\*\* 로 변경 — 쿼리에서 측정된 AP 차원으로만 euclidean 계산, 누락 AP 는 거리에서 제외 (sklearn KNeighborsClassifier → numpy 직접 구현)

\- 학습 데이터는 기존 평균(centroid, `load\_training\_set`) 그대로 유지 → `db/fingerprint.py` 는 원복 (변경 없음)

\- `K` 하드코딩(3) 제거 → `config.KNN\_K` 사용

\- 측위 방법 4종(평균 / raw / multi-centroid / 마스킹) 비교 검증 (로컬 스크립트)

\- dropout 시뮬레이션 실측: 출퇴근 혼잡 21% → 87.5% (+66%p), 극심 9.9% → 73.2%



\## 왜 이렇게 했는지

\- \*\*dropout 대응\*\*: 지하철역은 사람/기둥에 막혀 AP 가 자주 누락됨. 기존은 누락 AP 를 -100 으로 채워 거리를 왜곡('측정 안 됨'을 '멀다'로 처리) → 오인식. 마스킹은 잡힌 AP 로만 비교해 AP 가 막혀도 정확.

\- \*\*raw 폐기\*\*: raw 개별 학습도 시도했으나 held-out + dropout 검증 결과 가혹한 환경에서 평균보다 낮았음(보통 혼잡 -6.8%p). multi-centroid 도 유사. 마스킹만 압도적이라 채택.

\- \*\*학습은 평균 유지\*\*: 평균 centroid 가 측정 노이즈를 이미 상쇄해 안정적. 예측만 마스킹으로 바꿔 두 장점을 결합.



\## 막힌 것 / 결정 미뤄진 것

\- 깨끗한 저장 데이터로는 평균·raw·마스킹 모두 100% (held-out 포함) — 차이는 dropout 시뮬레이션에서만 드러남. 실제 현장 신규 측정으로 추가 검증 필요.
